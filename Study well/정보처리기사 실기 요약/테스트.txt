정적테스트
프로그램 실행 X

워크스루, 인스펙션, 코드 검사


동료 검토: 작성자가 명세서 내용을 직접 설명. 이해관계자, 동료들이 설명을 들으면서 결함을 발견하는 형태

워크스루: 회의 전에 명세서를 배포하여 사전 검토한 후 결함 발견.

인스펙션: 작성자를 제외한 다른 검토 전문가들이 확인하면서 결함을 발견하는 형태



동적 테스트
프로그램 실행 O

화이트박스 테스트, 

종류 
기초 경로 검사(Base Path)
, 제어 구조 검사 Control Structure
(조건 검사(Condition), 루프 검사,(Loop) 데이터 흐름 검사(Data Flow))

검증 기준
문장 검증 기준  모든 구문이 한번이상 수행   - Statement Coverage
분기 검증 기준  모든 조건문이 한번이상 수행  - Branch Coverage, Decision coverage
조건 검증 기준  전체 조건문에 상관없이 개별 조건문(TRUE,FALSE)인 경우 수행  - Condition Coverage
분기/조건 기준  - Branch/Condition Coverage





블랙박스 테스트   


동치 분할 검사(Equivalence Partitioning Testing)
경계값 분석(Bounday Value Analysis)
원인-효과 그래프 검사(Cause-Effect Graphing Testing)
오류 예측 검사(Error Guessing)
비교 검사(Comparison Testing)




개발 단계에 따른 분류
코딩 직후
-> 단위 테스트(화이트,블랙)  모듈 확인
->  통합 테스트 (상향식, 하향식, 혼합(샌드위치), 빅뱅 , 백본, 회귀 시험)인터페이스 확인  

-> 시스템 테스트 (기능,비기능)    기능 확인    
-> 인수 테스트(알파, 베타)    요구사항 확인

회귀 테스트(regression) - 소프트웨어의 변경 또는 수정된 코드에 새로운 결함이 없음을 확인하는 테스트


빅뱅 테스트 : 드라이버/스텁 없이 실제 모듈로 테스트 

백본 테스트 : 드라이버/스텁을 필요에 따라 만들어 사용하는 방식



샌드위치(혼합식 테스트)
상향식과 하향식의 장점을 이용하는 방식
